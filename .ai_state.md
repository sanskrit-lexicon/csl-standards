# Project Objective: CSL Standards — TEI/OntoLex interoperability workbench for CDSL (MW-PWG-PWK)

Status date: 2026-06-20.

A research instrument that maps CDSL dictionary records into TEI (archival) and
OntoLex/FrAC (semantic) profiles, tracks every loss as an evidence-bound report,
and publishes the analysis. Pipeline: `npm run build-pilot` (sample -> neutral
model -> kosa parse -> loss reports -> exporters -> validators -> analyze).

## Dev Notes & Hypotheses

- [2026-06-20] `npm run scale-check` is non-invasive by design: it snapshots
  `data/pilot` and `src/data/pilot`, runs 500/1000 pilots, captures compact
  summaries under `data/scale`, restores the canonical 250-case pilot files, and
  fails if those directories are left dirty.
- [2026-06-20] Scale results: central asymmetry holds at 500/1000. TEI has 0
  lossy western-dictionary reports, OntoLex has 0 clean reports, and lossy burden
  remains neutral > OntoLex > TEI. Evidence-loss share stays about 70%.
- [2026-06-20] Salt API Phase 0 is a `csl-standards` contract package only:
  normative profile, RU mirror, OpenAPI, GraphQL SDL, loss report, and handoff
  checklist. No server implementation belongs here.
- [2026-06-20] `csl:` vocabulary packaging now exists as both docs and site page:
  term index plus loss phenomenon -> construct -> validation-rule crosswalk.
