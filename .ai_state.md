# Project Objective: CSL Standards ‚Äî TEI/OntoLex interoperability workbench for CDSL (MW-PWG-PWK)

A research instrument that maps CDSL dictionary records into TEI (archival) and
OntoLex/FrAC (semantic) profiles, tracks every loss as an evidence-bound report,
and publishes the analysis. Pipeline: `npm run build-pilot` (sample ‚Üí neutral
model ‚Üí ko≈õa parse ‚Üí loss reports ‚Üí exporters ‚Üí validators ‚Üí analyze).

## üßÝ Dev Notes & Hypotheses (Bugs, ideas, context)

- **Reproducibility**: generators honor `SOURCE_DATE_EPOCH` and otherwise omit
  `generatedAt` (scripts/lib/provenance.mjs), so `build-pilot` is byte-stable ‚Äî
  no more timestamp churn.
- **Shared libs**: `scripts/lib/citations.mjs` (the `<ls>` parser, used by
  build-neutral + both exporters) and `scripts/lib/mw-senses.mjs` (MW sense
  segmenter). The neutral model is the canonical citation layer; archival TEI and
  OntoLex still re-extract independently (intentional ‚Äî they keep a higher cap).
- **Committing churn pitfall**: `git checkout -- .` to clean EOL churn will revert
  *uncommitted* edits. Always commit real work *before* any cleanup; stage real
  files explicitly and skip EOL-only artifact churn.
- **Sampler invariants** (learned at 250): require a counterpart in all three
  dictionaries (drop MW-only); detect `hasHedge` on the *stored/compacted* raw so
  the phenomenon matches the truncated artifacts. AP90 is attached *after*
  selection (not scored), so adding a 4th dictionary does not change which cases are
  chosen ‚Äî keeps artifact churn additive.
- **Adding a dictionary** is now a one-line edit in `scripts/lib/dictionaries.mjs`
  (OPTIONAL_DICTS + DICT_LABEL/DIR/FILE). An optional dict attaches per case when it
  shares the `<k1>` headword (sampler matches after selection, so the 250 chosen
  cases never change) and is woven across **all four profiles**: OntoLex (csl:
  SourceRecord + attestations), archival TEI (presentDicts ‚Üí source-entry/taxonomy/
  citations), the loss corpus (evidence-class reports via emitEvidenceClasses), and
  Lex-0 (entry-level citations + sourceDesc listBibl). The tri-dict (mw/pwg/pwk) is
  still the case-id prefix (`mw-pwg-pwk:key`) and artifact-filename stem, and is the
  required backbone in the validators; optional dicts are additive on top. A dict
  with no `<ls>` apparatus (FRI, a reader) adds source records but no citations/loss
  reports. The PWG‚ÜíPWK‚ÜíMW lineage family stays tri-dict-specific.
- **Central finding holds at 250**: TEI never lossy for Western cases, OntoLex
  never clean, the neutral lineage lane most lossy. Regenerate the analysis with
  `npm run analyze-loss`; full prose in docs/LOSS_ANALYSIS.md.
- `.claude/` (harness state) and `.github/scripts/__pycache__/` are gitignored /
  untracked ‚Äî not part of the project.
- **Loss-report coverage closed**: `build-loss-reports.mjs` emits an
  evidence-class sub-typing family (`named-kosha-citation`, `citation-coordinate`,
  `editorial-reference`; all `target: ontolex`, `model-vocabulary-gap`) + a
  `data-quality`/`source-anomaly` report for `[sic]`. `analyze-loss` now reports
  `schemaPhenomenaNotEmitted: []` + `roadmapCausesNotEmitted: []`; only the
  `failure` *status* stays (legitimately) unused. Detectors are curated heuristics
  (ko≈õa sigla, editorial abbrevs, coordinate regex, literal `[sic]`) ‚Üí counts are a
  lower bound. `data-quality` added to the schema enum.
- **#29 left `ontolex-review.json` stale**: the container-refinement PR regenerated
  the individual `ontolex/*.json` graphs (83 nodes each) but not the review summary
  (still at 68-node, pre-#29 counts); this slice's `build-pilot` resynced it. When
  a PR changes an exporter, regenerate *all* derived review artifacts, not just the
  primary outputs.
