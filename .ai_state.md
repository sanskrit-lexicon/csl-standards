# Project Objective: CSL Standards — TEI/OntoLex interoperability workbench for CDSL (MW-PWG-PWK)

A research instrument that maps CDSL dictionary records into TEI (archival) and
OntoLex/FrAC (semantic) profiles, tracks every loss as an evidence-bound report,
and publishes the analysis. Pipeline: `npm run build-pilot` (sample → neutral
model → kośa parse → loss reports → exporters → validators → analyze).

## 🧠 Dev Notes & Hypotheses (Bugs, ideas, context)

- **Reproducibility**: generators honor `SOURCE_DATE_EPOCH` and otherwise omit
  `generatedAt` (scripts/lib/provenance.mjs), so `build-pilot` is byte-stable —
  no more timestamp churn.
- **Shared libs**: `scripts/lib/citations.mjs` (the `<ls>` parser, used by
  build-neutral + both exporters) and `scripts/lib/mw-senses.mjs` (MW sense
  segmenter). The neutral model is the canonical citation layer; archival TEI and
  OntoLex still re-extract independently (intentional — they keep a higher cap).
- **Committing churn pitfall**: `git checkout -- .` to clean EOL churn will revert
  *uncommitted* edits. Always commit real work *before* any cleanup; stage real
  files explicitly and skip EOL-only artifact churn.
- **Sampler invariants** (learned at 250): require a counterpart in all three
  dictionaries (drop MW-only); detect `hasHedge` on the *stored/compacted* raw so
  the phenomenon matches the truncated artifacts.
- **Central finding holds at 250**: TEI never lossy for Western cases, OntoLex
  never clean, the neutral lineage lane most lossy. Regenerate the analysis with
  `npm run analyze-loss`; full prose in docs/LOSS_ANALYSIS.md.
- `.claude/` (harness state) and `.github/scripts/__pycache__/` are gitignored /
  untracked — not part of the project.
